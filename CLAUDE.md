## Hard Rules -- Never Break These

### 1. Script size limit

**100,000 compiled characters maximum.** This is a hard SE engine limit.
Always check total character count before presenting a final script.

### 2. Read the file from disk before patching

**Always read the current file from disk before any edit.** Never patch from
memory, from a previous response, or from a stale `/tmp` copy. If the file was
written since the last read, re-read it first.

### 3. Do not start coding until told to

Do not begin implementing a feature or writing code unless RevGamer explicitly
says to start. Design and planning discussions first; code when instructed.

### 4. Complete scripts only

Never provide placeholders, omitted sections, pseudocode, or
"// unchanged code here" comments. Every presented script must be complete
and ready to paste.

### 5. Plain ASCII only in string literals

No em dashes (--), smart quotes, arrows, or any character above ASCII 127
inside string literals. These cause minifier compile errors. Comments may
contain non-ASCII; string literals must not.

### 6. No static readonly fields

`static readonly` field declarations are not allowed in SE PB scripts -- they
cause compile errors. Use plain `static` fields (reassigned each update) or
instance fields instead. `static` methods are fine.

### 7. No C# 7+ syntax

SE uses an older Roslyn version. The following will not compile:
- Inline `out` variable declarations: use `Type name; TryParse(..., out name);`
- String interpolation `$"..."`: use `string.Format(...)` or concatenation
- Tuples `(string a, int b)`: use a class or struct
- `?.` null-conditional in complex chains: test carefully, it can work in simple cases

### 8. No ModAPI, no using statements, no class wrapper

Do not include `namespace`, `partial class Program`, or `using` statements.
Do not use `MyAPIGateway` -- it is ModAPI only. The game injects the fixed
using set automatically. Paste class body members only.

## File Workflow

### Standard edit workflow

1. `Filesystem:copy_file_user_to_claude` to pull current disk version into Claude's workspace
2. Patch using Python `content.replace(old, new, 1)` for targeted changes,
   or line-number slicing when string matching fails due to whitespace
3. Validate brace balance with a comment/string-aware Python parser before writing
4. Write large files in chunks via `Desktop Commander:write_file` (rewrite + append)
   rather than `Filesystem:write_file` which truncates on large content
5. Copy final output to `/mnt/user-data/outputs/` and call `present_files`

### AGM v2.0 minification

Minifier: `F:\Space Engineers Script\minifiedtool\IngameScriptMergeTool\IngameScriptMergeTool.exe`
Flags: `-s [sln path] -m -d "[script name]"`

### PowerShell execution

Always write a `.ps1` file to `C:\Users\corra\AppData\Local\Temp\` and run with:
```
powershell.exe -NoProfile -ExecutionPolicy Bypass -File "C:\...\script.ps1"
```
Do not use inline `-Command` with `$variable` syntax -- the `$` gets stripped
by cmd's quoting layer and causes silent failures.

### Python patching

Run via `bash_tool`: `python3 patch_script.py`
Or on the user's machine: `cmd /c "py C:\path\to\patch.py"`

## Tag Style

Always use `[bracket]` style tags. Never use `!prefix` or `{brace}` style
(except legacy AGM v1 connector tags which used `{brace}`).

Examples: `[HogOS-Dock]`, `[no-agm]`, `[locked]`, `[assemble-only]`

---

## AGM v2.0 Architecture Notes

- Single-PB, bracket-tag style, 19-step pipeline
- `[no-agm]` on a connector excludes the entire docked construct including
  subgrids via `IsSameConstructAs`
- All turrets (`IMyUserControllableGun`) are excluded from sort sources
- `[locked]` does NOT block dock grid exclusion
- `autoDisasm=false` by default; `D` suffix in autocraft template enables
  per-item disassembly; RevGamer edits `autoDisasm` manually
- `[assemble-only]` and `[disassemble-only]` are block name tags, not Custom Data
- `[Stock]` redistributes existing items -- it cannot create items from nothing

---

## HogOS v2.0 Architecture Notes

- Screens: Splash (auto dock-status when `[HogOS-Dock]` connector attached),
  Loading (immersive boot), Power, OreCargo, Weight
- Logo: `Textures\FactionLogo\Miners\MinerIcon_3.dds`
- Tag a connector name with `[HogOS-Dock]` to enable dock-aware Splash
- Utility/Drills screens and DrillManager removed in v2.0

  ```
  [HogOS]
  Surface0=Splash
  Surface1=Power
  Surface2=OreCargo
  Surface3=Weight
  ```

## SE API Hard Limits (Confirmed by Testing)

| Issue | Rule |
|---|---|
| `IMyLargeTurretBase.Azimuth`/`.Elevation` | Write-only. Reading always returns 0. Setting them breaks turret AI permanently -- park mode must only set `EnableIdleRotation = false` |
| `static readonly` fields | Not allowed in PB scripts -- compile error |
| Field-level `new Color()` initializers | Not allowed at field level -- must be inside a method |
| `TrackTarget()` | Entity priority hint only, not a forced aim method |
| `VentStatus` leak detection | Correct enum: `VentStatus.Depressurized` |
| `IsSameConstructAs` | Correct method for docked grid identification |
| Corner LCDs | Both `IMyLightingBlock` and `IMyTextSurfaceProvider` -- do not gate with `if (light == null)` |

## TypeId Rules

Block TypeIds and item TypeIds are not interchangeable.

| Category | Full runtime form | Example |
|---|---|---|
| Block definition | `MyObjectBuilder_CargoContainer` | For block lookups |
| Ore | `MyObjectBuilder_Ore` | For inventory items |
| Ingot | `MyObjectBuilder_Ingot` | For inventory items |
| Component | `MyObjectBuilder_Component` | For inventory items |
| Ammo | `MyObjectBuilder_AmmoMagazine` | For inventory items |

- Verify every TypeId/SubtypeId pair against the reference scripts or game
  data at `C:\Program Files (x86)\Steam\steamapps\common\SpaceEngineers\Content\Data\`
- Treat any unknown or ambiguous TypeId as an error until verified
- Read `SE_BlockTypeIds.md` and `SE_ItemTypeIds.md` before finalizing TypeId logic

## Mandatory Verification Before Presenting Any Script

1. Check every `{}`, `()`, `[]` pair is balanced using a comment/string-aware parser
2. Check every referenced variable, method, class, and interface is defined
3. Check method signatures, argument types, and return types
4. Check for non-ASCII characters in string literals
5. Check for `static readonly` field declarations
6. Check for C# 7+ syntax
7. Check all TypeId/SubtypeId pairs
8. Verify total character count is under 100,000
9. Trace the main execution paths and fix any errors found

## DDS Texture Verification

To identify or verify sprite texture paths from the SE game install:

1. Locate `.dds` files under:
   `C:\Program Files (x86)\Steam\steamapps\common\SpaceEngineers\Content\`
2. Convert to PNG using SE's bundled tool:
   `C:\Program Files (x86)\Steam\steamapps\common\SpaceEngineers\Tools\TexturePacking\Tools\texconv.exe`
3. View the rendered PNG via `Desktop Commander:read_file`

This is the only reliable way to identify sprite names -- GitHub raw file
access is unreliable for SE assets.

## Performance Guidelines

- Cache block lists; refresh them deliberately (not every tick)
- Reuse `List<T>` fields with `.Clear()` before filling
- Single raycasts only; cache results
- No expensive per-tick calls on large grids
- Spread heavy work across update cycles when needed
- Use `IsSameConstructAs(Me)` to exclude subgrids where appropriate
